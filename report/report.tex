\section{Introduction \& Problem Definition}
Traditional real-estate valuation relies heavily on either manual appraisal (which is slow and subjective) or standard regression models (which provide numbers without context). For an investor, a "number" is insufficient; they require reasoning, market trends, and risk analysis.

This project introduces an \textbf{Agentic AI Assistant} that bridges the gap between numerical regression and cognitive reasoning. The primary goal is to build an end-to-end system that predicts a property's market value and immediately synthesizes a professional advisory report based on real-time market data.

\section{System Architecture}
The system employs a multi-layered architecture coordinated by a \textbf{LangGraph} state machine. Unlike standard linear scripts, this agentic approach allows for modular, stateful execution of disparate AI tools.

\subsection{The 3-Step Agentic Graph}
\begin{enumerate}
    \item \textbf{ML Prediction Node}: Utilizes a Random Forest Regressor to provide a quantitative baseline.
    \item \textbf{RAG Retrieval Node}: Extracts qualitative market "knowledge" from a vectorized textual database in response to the specific property attributes.
    \item \textbf{GenAI Synthesis Node}: A Large Language Model (Llama 3 via Groq) analyzes the outputs of the previous two nodes to generate a human-readable advisory report.
\end{enumerate}

\section{Technical Implementation}

\subsection{Machine Learning Model (Price Prediction)}
While the agent provides the "brain," the ML model provides the "facts." We trained a \textbf{Random Forest Regressor} on 417,561 observations using 13 engineered features, including:
\begin{itemize}
    \item \textbf{Geospatial}: Haversine distance to Central London.
    \item \textbf{Physical}: Floor Area, Room Density (Sqm/Room).
    \item \textbf{Environmental}: EPC Energy Ratings (ordinal mapping).
\end{itemize}

\subsection{Retrieval-Augmented Generation (RAG)}
To provide non-hallucinated advice, we implemented a RAG system. We utilize a curated knowledge base of London market trends. When a user queries a property, the system performs a localized search to find specific risks (e.g., leasehold issues) or opportunities (e.g., gentrification trends) relevant to that property type and location.

\subsection{Hallucination Mitigation Strategy}
Ensuring clinical accuracy in real-estate advice is critical. Our system handles hallucinations through a three-tier robustness strategy:
\begin{enumerate}
    \item \textbf{Grounding via RAG}: By providing the model with a fixed context from the knowledge base, we reduce the LLM's reliance on its internal "stale" training data.
    \item \textbf{Negative Constraint Prompting}: The system prompt explicitly forbids the agent from "guessing" or "assuming" facts. The instructions command the agent to state uncertainty clearly if the provided market insights are insufficient.
    \item \textbf{Chain-of-Context Verification}: The LangGraph architecture ensures that the "Advice" node can only proceed once the "Retrieval" node has passed its specific context, creating a verified data pipeline.
\end{enumerate}

\subsection{Inference Optimization with Groq}
To satisfy the rubric's "Stability" and "Performance" requirements, we utilized the \textbf{Groq LPU} infrastructure. By using the \texttt{llama-3.3-70b-versatile} model, we achieve near-instantaneous synthesis of complex real-estate reports, ensuring a professional user experience.
